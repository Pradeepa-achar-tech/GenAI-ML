              explain: 'An **edge** is a place where image brightness changes sharply — the boundary between two objects, the outline of a coin, the edge of a road. **Canny edge detection** (John F. Canny, 1986) is the gold-standard algorithm because it does FOUR things in sequence and does each one well: (1) Gaussian-blur the image to suppress noise. (2) Compute the brightness **gradient** at every pixel using **Sobel kernels** — gradient magnitude tells you HOW strong an edge is, gradient direction tells you which way it runs. (3) **Non-maximum suppression** thins thick gradient ridges down to 1-pixel-wide lines by keeping only local maxima along the gradient direction. (4) **Double threshold with hysteresis** keeps any pixel above `high` as a definite edge, drops any pixel below `low`, and keeps middle pixels ONLY if they connect to a definite edge. The result: clean, thin, connected edge pixels with very few false positives — exactly what you want before contour finding, line detection, or feature extraction. One line in OpenCV: `cv2.Canny(img, low, high)`.',
              analogy: 'Think of Canny as a **four-stage Indian railway ticket inspector** working a single coach.\n\n**Stage 1 — the cleaner**. That direction is critical because real edges are perpendicular to the steepest gradient, and the inspector wants to walk along the edge, not across it. After the surveyor, the gradient image looks like a thick, fuzzy outline — many adjacent pixels all reporting "I am near a step".               theory: '**Sobel gradients** — the engine under Canny. Two kernels:\n```\nSobel_x = [[-1, 0, +1],          Sobel_y = [[-1, -2, -1],\n           [-2, 0, +2],                     [ 0,  0,  0],\n           [-1, 0, +1]]                     [+1, +2, +1]]\n```\nApply each as a convolution → `Gx` (horizontal change) and `Gy` (vertical change). Then:\n• **Magnitude**: `G = sqrt(Gx² + Gy²)` (or the cheap `|Gx| + |Gy|`).\n• **Direction**: `θ = atan2(Gy, Gx)`. Canny rounds θ to one of four bins (0°, 45°, 90°, 135°) before NMS.\n\n**`cv2.Canny(image, threshold1, threshold2, L2gradient=False, apertureSize=3)`**:\n• `threshold1` = low (hysteresis lower bound).\n• `threshold2` = high (hysteresis upper bound).\n• `L2gradient=True` uses true Euclidean magnitude (slightly slower, slightly cleaner).\n• `apertureSize` is the Sobel kernel size; default 3 is almost always right.\n\n**Choosing thresholds — three common heuristics**:\n1. **Median-based (Adrian Rosebrock heuristic)**: compute `med = np.median(blurred)`; set `low = max(0, 0.66 * med)`, `high = min(255, 1.33 * med)`. Adapts to image brightness.\n2. **Otsu-based**: run Otsu thresholding to get T_otsu; set `high = T_otsu`, `low = 0.5 * T_otsu`.\n3. **Manual sweep**: try `(50, 150)`, `(100, 200)`, `(150, 250)` and visually pick.\n\nThe **1:2 or 1:3 ratio** between low and high is the classic Canny recommendation.\n\n**What Canny is NOT good for**:\n• Very textured images (grass, sand, fur) — every texel becomes an edge. Use a learned edge detector (HED, BDCN) instead.\n• Subpixel-precision needs — Canny is pixel-resolution. For sub-pixel use Sobel + interpolation.\n• Coloured edges (e.g. red text on green background of equal brightness) — Canny works on luminance only. Convert via Y of YCrCb or run Canny per channel and OR-combine.\n\n**What you do AFTER Canny — the typical pipeline**:\n1. **`cv2.findContours(edges, ...)`** — turn pixel edges into ordered point lists.\n2. **`cv2.HoughLinesP(edges, ...)`** — detect straight lines (lane detection, document edges).\n3. **`cv2.HoughCircles(...)`** — detect circles (coins, pupils, traffic signs).\n4. **`cv2.approxPolyDP(contour, eps, closed=True)`** — simplify contours into polygons (find the 4 corners of a document for perspective correction).\n\n**Other classical edge detectors** (mentioned in interviews):\n• **Sobel** — raw gradient; thick edges, no NMS. Use as the first step inside Canny or for direction-only analysis.\n• **Laplacian** (`cv2.Laplacian`) — second derivative; sensitive to noise but catches both directions of change at once.\n• **Scharr** — Sobel variant with better rotational symmetry; use `cv2.Sobel(img, ddepth, dx=1, dy=0, ksize=cv2.FILTER_SCHARR)`.\n• **Prewitt** — older, simpler than Sobel; mostly historical.',
